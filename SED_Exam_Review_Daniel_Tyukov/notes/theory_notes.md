## 1. The design problem

A1 converts the magnetic field picked up by a coil into an audio voltage for an ADC. It is a
**voltage–voltage integrator**, because the coil is a differentiator and the cascade must be
flat over the audio band.

| Spec | Value | What it sets |
|---|---|---|
| Transfer (integrator) | H(s) = (1/τᵢ)/s = 62.4×10³/s | gain, RꜰCᵢ product |
| Integration constant | τᵢ = 16.06 µs | RꜰCᵢ |
| Audio band | fₘᵢₙ = 600 Hz … fₘₐₓ = 6 kHz | bypass Rᵦ, bandwidth |
| Output noise (DIN-A) | V_o,noise ≤ 10.6 µV RMS (≈ 30 dBSPL) | first-stage gₘ, cᵢₛₛ |
| Load | C_L = 10 pF | stability, drive current |
| Supply / power | 1.1–1.3 V / ≤ 1 mW | bias currents |
| Coil | L_s = 120 mH, R_s = 875 Ω, C_s = 9.4 pF | source noise, R_t |
| Coil sensitivity | −59.4 dBV/(A/m) @ 1 kHz | required gain |
| Reference frequency | f_ref = 1 kHz | gain matching (mic = loop) |

## 2. Part 1 — Specification & environment

### 2.1 The pick-up coil is a differentiator
Induced voltage is proportional to the rate of change of flux:

  v_coil(t) = −N · dΦ/dt ∝ dB/dt

So A1 must **integrate** to recover a flat audio response.

### 2.2 Coil model and resonance
Small-signal coil + cable: series L_s, R_s with parasitic C_s, plus termination R_t.
Self-resonance:

  f_res = 1 / (2π·√(L_s·C_s)) ≈ 1 / (2π·√(0.12 × 9.4×10⁻¹²)) ≈ 150 kHz

Undamped this peak rings and overloads the input. Adding R_t across the input gives
**critical damping** (Q = 1/√2 ≈ 0.707), R_t ≈ 80 kΩ. R_t then contributes thermal noise
(budgeted at ~9% of the total, via V_ns).

### 2.3 Output interface
- Source-select switch (mic / loop): needs V_mic = V_loop at f_ref ⇒ fixes the gain magnitude.
- ADC input is a voltage amplifier: Zᵢ → ∞, draws no current. Only load = parasitic C_L = 10 pF.
⇒ A1 is a **voltage–voltage amplifier driving a capacitor**, not a power stage.

### 2.4 DIN-A noise weighting
Noise is weighted by the perceptual DIN-A curve (0 dB at 1 kHz) before integrating to RMS:

  σ_o² = ∫₀^∞ |A(f)|² · S_o(f) df

This makes the noise budget perceptually meaningful (tight where hearing is sensitive).

## 3. Part 2 — Transfer & feedback configuration

### 3.1 Options (generate first, then choose)
- **Option A — voltage–voltage** (chosen): Z₁ = capacitor, Z₂ = resistor; integrator in
  feedback. Coil R_s enters only the *noise*, not the transfer.
- **Option B — current–voltage (transimpedance):** Z₂ = inductor, Z₁ = resistor; the inductor
  integrates. Coil R_s now enters the *transfer*, and the inductor is large (area, noise).

Decision: **Option A**, kept consistent throughout.

### 3.2 Asymptotic-gain (feedback) model
Accuracy follows the passive feedback network, not the transistor:

  A_f = A_∞ · [ L / (1 − L) ] + [ 1 / (1 − L) ]
                 └── servo ──┘   └── direct ──┘

- A_∞ = ideal nullor transfer = 1/(s·Rꜰ·Cᵢ)
- L = loop gain (return ratio), ≈ 45 dB at DC for the dual stage
- servo L/(1−L) → 1 when |L| ≫ 1 ⇒ A_f → A_∞; servo ≈ L when |L| ≪ 1
- direct 1/(1−L) = feed-through, negligible here

The Bode plot is the |·| slice (the 0-axis) of the Nyquist plot.

### 3.3 Sizing the feedback network
- Upper bound on Rꜰ from the **noise** budget (B_n1 = 0.4): Rꜰ,max ≈ 7.54 kΩ
- Lower bound on Rꜰ from the **power/current** budget (B_i1 = 0.1): Rꜰ,min ≈ 1.39 kΩ
- Chosen **Rꜰ = 3.24 kΩ**
- Cᵢ = τᵢ / Rꜰ = **4.96 nF**
- Bypass **Rᵦ = 1/(2π·Cᵢ·fₘᵢₙ) = 53.5 kΩ** — sets the low corner at fₘᵢₙ and gives DC bias a path.

## 4. Part 3 — First stage: noise design

### 4.1 Why the first stage owns the noise
Model the controller as a **noisy nullor** (ideal nullor + the input device's v_n, i_n). Later
stages' noise is divided by the first-stage gain, so it is negligible. The whole budget is spent
on stage 1 ⇒ design stage 1 from the noise spec first.

### 4.2 The knobs
Input-referred thermal noise of a CS stage:

  v̄_n² = 4kTγ / gₘ   [V²/Hz]

- ↑gₘ ⇒ ↓noise (more current for less voltage)
- wider device ⇒ ↑cᵢₛₛ ⇒ more input loading (bad)
- **weak inversion** ⇒ largest gₘ/I_D (transconductance efficiency)
- design target: minimise the product **gₘ·cᵢₛₛ**

### 4.3 Stage choice: CS vs CG vs CD
The CS chain (ABCD) matrix has the **smallest** elements ⇒ most nullor-like ⇒ later stages add
little noise. CG and CD are followers (no voltage gain). ⇒ **first stage = common-source**, in
saturation.

### 4.4 Budget → parameters
1. Find output noise spectrum S_o(f).
2. Integrate (DIN-A weighted) → variance σ_o² = (B_n2·V_o,noise)².
3. Express gₘ = f(cᵢₛₛ).
4. Lowest feasible cᵢₛₛ → gₘ, I_D, then W, L via the **EKV** model.

Result (PMOS input, gₘ·cᵢₛₛ optimum): gₘ ≈ 73 µS, W ≈ 19.1 µm, L ≈ 0.72 µm, I_D ≈ 8.1 µA.
(Alternative "min gₘ" point: gₘ ≈ 34 µS.)

## 5. Part 4 — Single stage check
A single CS has only 3 usable terminals ⇒ use a differential pair (≈4× current & area).
- Current drive into C_L: ✓
- Voltage headroom on 0.9 V: ✓
- Loop gain |L_DC| < 1 across the band: ✗ ⇒ servo cannot enforce the transfer ⇒ **poor accuracy**.
⇒ move to a **dual stage**.

---

## 6. Part 5 — Dual stage

### 6.1 Why dual
Five competing costs: noise, power, accuracy, stability, area. **First stage dominates noise;
second stage dominates loop gain.** Split the job.

### 6.2 Topology
- Stage 1: **PMOS differential pair** (low noise, high CMRR, matches the floating 2-terminal coil)
- Stage 2: **NMOS common-source** (high loop gain, high current-drive, low power, small area)

### 6.3 Diff-pair vs single CS (stage 1)
| Differential pair | Single-ended CS |
|---|---|
| + lower noise | + less area |
| + high CMRR | + lower power |
| + matches the 2-terminal coil | + simple |
| − more area | − allows CM interference |
| − more complex | − higher noise |

Coil is floating + noise is priority ⇒ **differential pair**.

### 6.4 Design parameters
- Input PMOS pair: W₁ ≈ 19.1 µm, L₁ ≈ 0.72 µm, I_D1 ≈ 8.07 µA (= 2× the half-circuit)
- Output NMOS CS: W₂ = 10 µm, L₂ = 0.18 µm (minimum length), I_D2 ≈ 16.5 µA
- Loop gain ≈ 45 dB at DC

Design parameters that matter: gₘ₁ (noise), gₘ₂ (extra loop-gain factor), Rꜰ (transfer),
g_o1 (first-stage output conductance).

## 7. Part 6 — EKV model & pole-zero

Swap ideal devices for the **EKV** small-signal model (real gₘ, g_o, all parasitic caps) ⇒
accurate gain/phase and the true poles/zeros.

**Closed-loop poles (numeric, from SLiCAP doPZ):**

| Pole | f_n | Q | Note |
|---|---|---|---|
| dominant | ≈ 602 Hz | — | integrator + bypass |
| pair | ≈ 150 kHz | ≈ 0.71 | well damped |
| pair | ≈ 51 MHz | **≈ 5.4** | severely underdamped → peaking |

All poles have Re < 0 ⇒ **stable**. The gain plot follows the poles/zeros, with visible peaking
from the high-Q pair. (5 poles, 3 zeros for the closed-loop gain; the loop-gain HF pair sits at
≈ 4.3 MHz.)

## 8. Part 7 — Frequency compensation (MFM)

### 8.1 Target
A pole pair with Q > 1/√2 peaks and rings. **Maximally-flat magnitude** (Butterworth) sets the
dominant pair to **Q = 1/√2 ≈ 0.707**: no peaking, fastest flat response, good phase margin.

### 8.2 Phantom-zero method
Add R_phz in series with the load capacitor → a **zero in the loop gain** without a real pole in
the signal path:

  z = −1 / (R_phz · C_L)

Chosen because it is the **lowest-cost** compensation: no noise source in the signal path,
negligible power/area, no compensation capacitor (no added energy storage).

### 8.3 Result
R_phz ≈ **0.70 kΩ** pulls the high-frequency pair from **Q ≈ 5.4 → 0.71** (MFM). Peak removed,
passband untouched, all poles still in the LHP (stable). Cost: one resistor.
